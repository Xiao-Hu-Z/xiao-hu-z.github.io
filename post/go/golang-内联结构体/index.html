Go语言基础之内联结构体

Posted by xiaohu on Thursday, November 4, 2021

Struct embeding

golang 中没有继承，但是其结构体内联的方式和继承很像，掌握内联结构体，实现面向对象

小王是一个很棒的strcut，他有一个field叫length，小王对自己的length非常满意，所以他还有专门炫耀自己的长度method： ShowLength()

type XiaoWang struct{
    length int
}

func NewXiaoWang(length int) XiaoWang{
    return XiaoWang{
        length:length,
    }
}

func (zj XiaoWang) ShowLength(){
    fmt.Printf("XiaoWang's length is %d\n",zj.length)
}

func main() {
    zj := NewXiaoWang(18)
    zj.ShowLength()
}

运行结果为：小王的length 为 18 ！嗯，还是很不错的

突然有一天，老王来了，它是一个以小王为内联的结构体，为了变强，吸取了小王的精华，具有小王的所有field 和 method

type XiaoWang struct {
	length int
}

type LaoWang struct {
	XiaoWang
}

func NewXiaoWang(length int) XiaoWang {
	return XiaoWang{
		length: length,
	}
}

func (zj XiaoWang) ShowLength() {
	fmt.Printf("XiaoWang's length is %d\n", zj.length)
}

func NewLaoWang(length int) LaoWang {
	bzj := LaoWang{}
	bzj.length = length * 2 // 长度翻倍
	return bzj
}

func main() {
	bzj := NewLaoWang(18)
	bzj.ShowLength() // 老王可以调用小王的method
}

运行结果为：XiaoWang的length已经变为了36！原来在NewLaoWang 中，对length进行了翻倍

XiaoWang's length is 36

老王不满足于仅仅炫耀他的length，于是重写了他的 ShowLength()

func (bzj LaoWang) ShowLength() {
	fmt.Printf("LaoWang's length is %d,And can use for a long time!\n", bzj.length)
}
//LaoWang's length is 36,And can use for a long time!

再次运行，现在的ShowLength() 覆盖了 原来的ShowLength(), LaoWang不仅length很长，而且还很持久！

变长后的老王更自信了，想进一步炫耀他的宽度

package main

import "fmt"

type XiaoWang struct {
	length int
}

type LaoWang struct {
	XiaoWang
	width int
}

func NewXiaoWang(length int) XiaoWang {
	return XiaoWang{
		length: length,
	}
}

func (zj XiaoWang) ShowLength() {
	fmt.Printf("XiaoWang's length is %d\n", zj.length)
}

func NewLaoWang(length int, width int) LaoWang {
	bzj := LaoWang{}
	bzj.length = length * 2 // 长度翻倍
	bzj.width = width
	return bzj
}

func (bzj LaoWang) ShowLength() {
	fmt.Printf("LaoWang's length is %d,And can use for a long time!\n", bzj.length)
}

func (bzj LaoWang) ShowWidth() {
	fmt.Printf("XiaoWang's width is %d\n", bzj.width)
}

func main() {
	bzj := NewLaoWang(18, 8)
	bzj.ShowLength() // 老王可以调用小王的method
	bzj.ShowWidth()
}

Good，太length太width，很棒！！！

总结：

结构体会“继承”内联结构体的字段和方法，如果如果同名，则会覆盖

互斥锁内联

在实际编程中，我们会经常用到 锁 sync.Mutex,一般的写法是这样：

type M struct {
    mutex *sync.Mutex
    someFiled string
}

func (m *M) process(){
    fmt.Println("Helle World!")
}

func main() {
    m := M{}
    m.mutex = &sync.Mutex{}
    m.mutex.Lock()
    m.process()
    m.mutex.Unlock()
}

通过内联我们可以简写为：

type M struct {
    sync.Mutex
    someFiled string
}

func (m *M) process(){
    fmt.Println("Helle World!")
}

func main() {
    m := M{}
    m.Lock()
    m.process()
    m.Unlock()
}
